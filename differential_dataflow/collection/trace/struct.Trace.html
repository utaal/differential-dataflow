A collection of values indexed by key and time.

Safety

For reasons of borrow checking, it is difficult to merge references using already-allocated memory. The method get_collection_using uses its heap argument to store references that should remain valid for as long as the Trace is valid, but I cannot convince Rust of this fact because only I know that once installed, differences are immutable. Please do not call get_collection_using with a heap argument that may out-live the Trace itself.
